## Contribution Policy

MIGE OS is proprietary commercial software. External contributions are not accepted at this time.

## Internal Development

For internal team members:

1. Create feature branch from `develop`
2. Make changes following coding standards
3. Ensure all tests pass
4. Submit pull request for review
5. At least one approval required
6. CI/CD must pass before merge

## Coding Standards

- TypeScript strict mode required
- All functions must have type annotations
- No `any` types allowed
- Follow existing code style
- Write tests for new features

## Pull Request Process

1. PR title must be descriptive
2. PR description must include:
   - Purpose of changes
   - Breaking changes
   - Test coverage
3. All checks must pass
4. At least one code review required
